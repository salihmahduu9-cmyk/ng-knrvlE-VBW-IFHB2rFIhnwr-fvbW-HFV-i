# إعداد عدّاد الإحصائيات (API)

الموقع فيه API يعدّ ثلاثة أشياء:
- **مرات التشفير** — كل ضغطة على "شفّر"
- **إجمالي الزوّار** — زوّار فريدين (عبر كوكي)
- **نشط الآن** — المستخدمين خلال آخر 5 دقائق

كل شي مخزّن في **Upstash Redis** (مجاني، متكامل مع Vercel).

## خطوات الربط (مرة وحدة)

### 1) ركّب Upstash Redis في مشروعك على Vercel
- ادخل مشروعك في Vercel → تبويب **Storage**
- اضغط **Create Database** → اختر **Upstash** → **Redis**
- اختر اسم ومنطقة (أقربها لك) → Create
- اضغط **Connect** واربطها بالمشروع

هذا تلقائياً يضيف متغيّرات البيئة:
`KV_REST_API_URL` و `KV_REST_API_TOKEN`
(الكود يقرأها تلقائياً، ويدعم أيضاً `UPSTASH_REDIS_REST_URL/TOKEN`)

### 2) أعد النشر
- بعد ربط القاعدة، روح Deployments → **Redeploy**
- خلاص! العدّادات بتشتغل وبتطلع تحت الموقع

## نقطة مهمة
- لازم يكون في مجلد المشروع `package.json` (موجود) عشان Vercel يثبّت مكتبة `@upstash/redis` تلقائياً.
- لو الأرقام طلعت `—` بعد النشر، معناه القاعدة مو مربوطة أو الـ Redeploy ما صار. تأكد من خطوة الربط وأعد النشر.

## نقاط نهائية الـ API
- `GET /api/stats` — يسجّل زيارة ويرجّع الأرقام
- `GET /api/stats?event=obfuscate` — يزيد عداد التشفير ويرجّع الأرقام

الرد JSON:
```json
{ "obfuscations": 123, "visitors": 45, "active": 3 }
```
